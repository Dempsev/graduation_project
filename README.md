# COAD: Phononic/Mechanical Metastructure Bandgap Workflow

## Project Goal

This repository is a graduation-project workflow for **bandgap-oriented design of phononic crystals / mechanical metastructures**. The intended research path is:

```text
structure generation
-> COMSOL finite-element simulation
-> band calculation
-> bandgap extraction
-> dataset construction
-> surrogate model training
-> optimization / inverse design
```

The current repository is mainly focused on the **front half of that pipeline**: generating structures, running COMSOL in batch, exporting eigenfrequency tables, and extracting bandgap statistics.

## Structure Representation

The project currently uses two complementary structure descriptions:

1. **Continuous boundary parameterization**
   Fourier-style boundary parameters are used in the COMSOL-side geometry workflow to control the global contour.
2. **Discrete local topology**
   The local topology comes from the `32 x 32` binary matrices generated by the snake workflow. These matrices are the practical source of the "discrete structure" used throughout preprocessing and batch simulation.

In the current codebase, the most explicit executable path is:

```text
snake state matrix
-> contour extraction
-> COMSOL geometry rebuild
-> eigenfrequency / tbl1 export
-> bandgap postprocess
```

## Current Repository Status

The repository already contains a working skeleton for:

- generating snake states in batch
- converting binary matrices into contour point CSV files
- rebuilding COMSOL models from those contours
- running study/batch compute and exporting `tbl1`
- summarizing bandgap candidates from exported CSV files
- plotting bandgap summaries and per-case band diagrams

What is **not finished yet** is the fully standardized data pipeline for machine learning, including stable per-sample packaging, dataset assembly, and surrogate-model training.

## Main Entry Points

### 1. Generate snake states

```bash
python snake/generate_states.py --episodes 200 --max-steps 500 --n 32 --agent q
```

Outputs are written under [`data/snake_state_matrices/`](/d:/graduation_project/coad/data/snake_state_matrices).

### 2. Convert matrices to contour points

```bash
python preprocess/main.py --dir data/snake_state_matrices --sample 0 --preview 1
```

Outputs are written under [`data/shape_contours/`](/d:/graduation_project/coad/data/shape_contours) and [`data/shape_contour_previews/`](/d:/graduation_project/coad/data/shape_contour_previews).

### 3. Batch-run COMSOL

```matlab
cd('d:\graduation_project\coad\runners');
run('run_shape_batch.m');
```

Important notes for the current runner:

- `startIndex` controls where the batch begins.
- `maxCount = 8` runs 8 cases by default. Set it to `0` to process all available shapes.
- `saveModel` controls whether `.mph` files are kept after export.

Exports are written to [`data/comsol_batch/tbl1_exports/`](/d:/graduation_project/coad/data/comsol_batch/tbl1_exports).

### 4. Postprocess bandgaps

```bash
python postprocess/analyze_bandgaps.py
python postprocess/plot_bandgap_summary.py
```

Outputs are written to [`data/postprocess_out/`](/d:/graduation_project/coad/data/postprocess_out).

## Batch Behavior

- Discrete perturbation samples that do not satisfy the geometry-contact rule are skipped.
- Skip records are written to [`data/comsol_batch/perturb_skip_log.csv`](/d:/graduation_project/coad/data/comsol_batch/perturb_skip_log.csv).
- Runtime failures are appended to [`data/comsol_batch/logs/run_shape_batch_errors.csv`](/d:/graduation_project/coad/data/comsol_batch/logs/run_shape_batch_errors.csv).
- Periodic boundary selection is based on rectangle position, not fixed boundary IDs.
- Legacy COMSOL scripts are preserved in [`runners/legacy_run/`](/d:/graduation_project/coad/runners/legacy_run).

## Next Development Direction

The next stage of the project should turn the current script collection into a stable dataset pipeline:

1. standardize each sample's saved artifacts
2. batch-generate a usable number of structures and simulations
3. store bandgap targets in machine-learning-friendly format
4. build the first `structure -> bandgap` dataset
5. train a baseline MLP surrogate model
6. connect the surrogate model to optimization search

At this stage, the priority is still **pipeline stability and data accumulation**, not model sophistication.
